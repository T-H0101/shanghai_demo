项目概览

统一光盘库管理平台方案概述

为什么做统一平台？

🎯 多站点管理痛点

各站点独立运维，数据分散，无法统一监控和汇总分析

📊 数据孤岛问题

站点间数据无法互通，跨站点查询和统计困难

⚡ 运维效率低

需要登录多个系统操作，切换成本高，容易出错

🔒 统一管控需求

需要统一的权限管理、审计日志和告警通知

增量同步策略

-- Task table incremental sync
SELECT id, uuid, task_type, task_name, status, burn_status,
       total_files, total_size, create_dt, update_dt
FROM tbl_task
WHERE update_dt > '{{last_sync_time}}'
ORDER BY id;

-- Device table incremental sync
SELECT lib_id, name, device_status, type, IP, mags, slots
FROM tbl_disc_lib
WHERE update_dt > '{{last_sync_time}}';

-- Alert table (only unsynced)
SELECT id, warning_type, warning_level, warning_msg,
       create_time, deal_status
FROM tbl_early_warning
WHERE create_time > '{{last_sync_time}}'
  AND deal_status = 0;

大表处理策略

tbl_file / tbl_folder 等大表暂不同步

识别的大表

表名预估数据量说明
tbl_file千万~亿级文件主表
tbl_folder千万级文件夹表
tbl_zip_file千万级压缩包文件表
tbl_sys_log百万~千万级系统日志表

为什么暂不同步？

📦 数据量过大

单站点 tbl_file 可能达千万条，多站点合并后可达亿级，增量同步压力巨大

💰 同步价值低

文件内容无需跨平台展示，前端 Demo 只需统计数据，明细数据按需查询即可

🗄️ 存储成本高

二进制数据占用大量空间，增量同步带宽消耗大，存储成本线性增长

⚙️ 技术复杂

大表同步需要分布式处理、分片策略、增量算法，实现成本高

多站点ID策略

source_site_id + source_id 复合主键方案

问题背景

多站点数据库同步时，不同站点的原始 ID 会产生冲突：

站点tbl_task.idtbl_disc_lib.lib_id
Site A1, 2, 3...1, 2, 3...
Site B1, 2, 3...1, 2, 3...
Site C1, 2, 3...1, 2, 3...

解决方案：复合主键 + 统一 ID

-- Unified platform table structure
CREATE TABLE unified_tasks (
    -- Unified ID (Primary Key)
    id BIGSERIAL PRIMARY KEY,

    -- Original ID info (for tracing)
    source_site_id VARCHAR(50) NOT NULL,  -- Source site ID
    source_table VARCHAR(100) NOT NULL,   -- Source table name
    source_id BIGINT NOT NULL,            -- Original primary key

    -- Business fields
    task_name VARCHAR(255),
    status INT,

    -- Metadata
    synced_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    -- Unique constraint (prevent duplicate sync)
    UNIQUE(source_site_id, source_table, source_id)
);

同步示例

字段Site A TaskSite B Task
id (unified)12
source_site_id'site_a''site_b'
source_id10011001
task_name'Backup Task-1''Archive Task-1'

两条记录共存，因为 (source_site_id, source_table, source_id) 组合唯一。
