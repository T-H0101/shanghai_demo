# 数据同步需求文档

> 创建时间：2026-05-19
> 依据：统一光盘库管理平台需求规格说明书

---

## 一、需求概述

**核心目标：** 同步各站点核心数据至统一系统，为后续跨站点检索、任务下发、盘笼移位提供数据支撑。

---

## 二、功能需求

### 2.1 同步范围

| 数据类型 | 说明 | 同步方式 |
|---------|------|---------|
| **设备信息** | 光盘库硬件节点、盘笼、盘位、光盘的状态/属性 | 实时同步 |
| **文件索引信息** | 所有光盘的文件列表、元数据 | 定时同步 |
| **权限信息** | 站点内账号/权限变更 | 实时同步 |
| **任务信息** | 任务状态/进度/结果 | 实时同步 |

**技术约束：** 同步数据需做增量过滤，仅同步变更数据，降低系统开销。

---

### 2.2 同步策略

| 策略 | 说明 | 触发条件 |
|------|------|---------|
| **实时同步** | 关键数据变更后立即同步 | 设备状态变化、任务状态变化、权限变更 |
| **定时同步** | 非关键数据按周期同步 | 可配置周期（默认15分钟） |
| **手动同步** | 管理员触发全量/增量同步 | 手动触发按钮 |

**同步失败处理：**
- 自动重试（可配置重试次数/间隔）
- 重试失败触发告警
- 支持手动重试

---

### 2.3 数据一致性校验

| 功能 | 说明 |
|------|------|
| 定时校验 | 每日定时校验统一系统与站点系统的数据一致性 |
| 差异报告 | 不一致数据生成差异报告，支持管理员手动修复 |
| 校验规则可配置 | 支持按数据类型（设备/索引/权限）分别校验 |

## 四、数据同步页面设计

### 4.1 站点同步状态（sites/page.tsx）

**站点卡片显示：**
- 同步状态 Badge：`已同步` / `同步中` / `同步失败` / `待同步`
- 最后同步时间
- 同步延迟（秒）

**站点操作：**
- 手动同步按钮
- 数据校验按钮

### 4.2 站点详情同步信息（sites/[id]/page.tsx）

**同步信息区域：**
```
同步状态: ● 已同步
最后同步: 2026-05-19 14:28:03
同步延迟: 8 秒
下一同步: 14:30:00 (2分钟后)
```

**同步操作：**
- 立即同步（手动触发）
- 数据校验（生成差异报告）

### 4.3 差异报告 Dialog

**触发条件：** 点击"数据校验"按钮

**显示内容：**
- 设备信息差异（数量/状态）
- 索引信息差异（文件数/校验码）
- 权限信息差异（账号/角色）
- 不一致数据详情

**操作按钮：**
- 同步差异（自动修复）
- 忽略（保持现状）
- 导出报告（生成 CSV/JSON）

## 五、数据同步配置

### 5.1 系统设置（app/settings/page.tsx）

**同步设置区域：**

| 配置项 | 字段路径 | 默认值 |
|--------|---------|--------|
| 实时同步 | `settings.sync.realtimeSync` | true |
| 定时同步周期(分钟) | `settings.sync.scheduledIntervalMinutes` | 15 |
| 全量同步时间 | `settings.sync.fullSyncTime` | 02:00 |
| 重试次数 | `settings.sync.retryCount` | 3 |
| 重试间隔(秒) | `settings.sync.retryIntervalSeconds` | 60 |
| 一致性校验时间 | `settings.sync.consistencyCheckTime` | 04:00 |
