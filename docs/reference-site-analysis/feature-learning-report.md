## 一、系统架构概览

### 1.1 系统定位
参考系统是**磁光电混合存储管理系统**，支持三种存储介质的统一管理：
- **磁存储**（硬盘）：高速在线存储
- **光存储**（光盘）：长期归档冷存储
- **电存储**（S3/缓存）：混合缓存层

### 1.2 功能模块划分

| 一级模块 | 二级模块 | 说明 |
|---------|---------|------|
| **存储管理** | 设备管理、存储卷管理、用户管理、缓存管理、预警管理 | 底层资源管理 |
| **数据备份** | 备份任务、计划任务、上传任务、复制任务、备份窗口、异地备份 | 数据备份功能 |
| **数据恢复** | 恢复任务、下载任务、一键导出 | 数据恢复功能 |
| **载体管理** | 载体数据浏览、光盘接收管理、批量复制、复制移交、介质检测、入库登记、出库管理 | 物理介质管理 |
| **任务管理** | 任务列表、任务设置、巡检任务 | 任务审计与配置 |
| **系统设置** | - | 系统配置 |

### 2.3 数据恢复模块

#### 2.3.1 恢复任务 (`数据恢复_恢复任务(从盘中恢复到服务器目标文件).png`)

| 项目 | 内容 |
|------|------|
| **系统标题** | 磁光电混合存储管理系统 |
| **当前模块** | 数据恢复 → 恢复任务 |
| **左侧数据树** | 已备份数据（根节点）→ 硬盘卷、光盘卷、外部存储设备 |
| **中部表格字段** | 名称、类型、大小、日期、操作 |
| **待恢复列表字段** | 名称、类型、大小、源路径、目标路径 |
| **恢复目标字段** | 服务器磁盘存储树（复选框选择目标文件夹） |
| **容量看板** | 目标路径、剩余容量、计划使用 |
| **检索选项** | 仅检索目录（复选框）、模糊检索（复选框） |
| **主要按钮** | 检索、视图切换、添加/删除目标、开始迁移 |

**核心业务逻辑**：
1. 用户在左侧数据树中选择存储卷
2. 浏览或搜索已备份数据
3. 将需要恢复的数据添加到待恢复列表
4. 选择恢复目标路径（服务器磁盘）
5. 核对容量后点击"开始迁移"

**业务洞察**：这是系统最核心的功能之一，实现从冷存储介质（光盘/硬盘）恢复数据到在线服务器。交互模式是"源→队→目→核→执"。

## 六、总结

### 6.1 核心差异

| 方面 | 参考系统 | 我们的Demo | 建议 |
|------|---------|-----------|------|
| **任务类型** | 备份/恢复/巡检/介质检测统一管理 | 基础任务列表 | 扩展类型覆盖 |
| **设备管理** | 磁光电网四类设备统一视图 | `/racks` 已有基础 | 完善字段展示 |
| **载体管理** | 槽位矩阵 + 详细介质信息 | 缺失 | 新增"载体"Tab |
| **数据恢复** | 源→队→目→核→执完整流程 | 缺失 | 新增恢复任务类型 |
| **介质检测** | RSER等物理指标 + 曲线图 | 缺失 | 新增检测功能 |

### 6.2 实施优先级

1. **Phase 1（P0）**：任务类型扩展 + 设备详情完善 + 载体数据浏览
2. **Phase 2（P1）**：数据恢复 + 介质检测 + 入库/出库管理
3. **Phase 3（P2）**：任务模板 + 批量复制 + 用户权限

### 6.3 关键原则

- **不新增独立页面**：所有功能整合到 `/tasks` 和 `/racks` 中
- **保持风格统一**：复用现有 UI 组件，保持企业后台风格
- **Mock数据优先**：先跑通 Mock 流程，再考虑后端接入
- **Tab切换为主**：多模块用 Tab 切换，避免页面爆炸
