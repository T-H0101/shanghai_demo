# 统一光盘库管理平台 - Demo 演示说明

> ⚠️ 本文档为 Demo 演示场景提供指引，说明如何展示已实现功能以及如何解释未实现的生产级能力。

---

## 1. 演示目标与边界

### 1.1 演示目标

- 展示统一光盘库管理平台的整体功能架构和交互逻辑
- 验证前端 UI 组件和页面联动是否符合需求设计
- 为后续真实后端接入提供可演示的原型基础

### 1.2 演示边界（Demo 局限）

| 类别 | Demo 现状 | 生产环境要求 |
|------|-----------|--------------|
| 数据 | Mock 数据，固定 6-8 条 | 真实数据库，动态数据 |
| 认证 | 所有页面直接访问，无登录 | ADFS/LDAP 单点登录 |
| 同步 | 模拟 Toast，无真实同步 | WebSocket 实时推送 |
| 任务 | 仅状态变更，无后端调度 | 后台任务队列 |
| 导出 | Toast 提示，无文件生成 | 真实文件下载 |
| 权限 | 仅 UI 保存，无 API | 权限同步到各站点 |

## 3. 各页面演示要点

### 3.1 首页仪表盘 `/`

**演示步骤**：
1. 展示 4 个统计卡片（站点总数、存储容量、任务进行中、告警数量）
2. 展示同步趋势图表（7 天趋势）
3. 展示站点健康热力图（6 个站点的健康状态）
4. 展示进行中任务表格（5 条任务，含进度条）
5. 展示告警中心（3 条最新告警）

**需要说明的 Mock 点**：
- 图表数据为静态生成，非实时数据
- 告警数量固定，不会自动更新

**引导话术**：
> "首页展示全局运营状态，包括站点健康度、任务进度、容量趋势。后续接入真实监控数据后，这些指标将实时更新。"

---

### 3.2 站点管理 `/sites`

**演示步骤**：
1. 展示站点列表（6 个站点，含状态、容量、同步时间）
2. 演示关键词搜索（输入"上海"过滤）
3. 演示状态下拉筛选（选择"离线"）
4. 点击"注册新站点"按钮，演示 Dialog 表单
5. 点击站点行内的"同步"按钮，演示 Toast 反馈
6. 点击站点行内的"SSO"按钮，演示 Toast 跳转提示
7. 点击站点行的"禁用"Power 按钮，演示状态变更

**需要说明的 Mock 点**：
- 新建站点仅添加到前端列表，不持久化
- 同步按钮无真实 API 调用
- SSO 跳转仅 Toast 提示，无真实跳转

**引导话术**：
> "站点管理支持多站点的集中配置，可以看到每个站点的状态、容量、同步时间。同步功能模拟了全量同步的交互流程。"

---

### 3.3 统一检索 `/search`

**演示步骤**：
1. 展示搜索筛选区域（站点、部门、文件类型、光盘编号等）
2. 输入关键词演示检索
3. 选择站点筛选，演示联动
4. 点击"检索"按钮，演示结果加载
5. 点击某行的"导出"按钮，演示 Toast 导出提示
6. 点击某行的"发起回迁"按钮，演示 Toast 回迁提示
7. 点击结果行的展开按钮，查看文件详情

**需要说明的 Mock 点**：
- 仅 8 条 Mock 数据，无法演示大规模检索
- 导出功能无真实文件生成
- 发起回迁无真实任务创建

**引导话术**：
> "统一检索支持跨站点的文件检索，可以通过站点、部门、文件类型、光盘编号等多维度筛选。这里的数据来自各个站点的索引同步。"

**演示步骤**：
1. 展示任务列表（7 个任务，4 种类型）
2. 点击不同类型 Tab（备份/回迁/校验/刻录）演示过滤

**需要说明的 Mock 点**：
- 新建任务仅添加到前端列表
- 操作（暂停/恢复/重试）无真实后端调度
- 日志为静态 Mock 数据

**引导话术**：
> "任务管理覆盖备份、回迁、校验、刻录等全生命周期。可以看到每个任务的状态、进度、操作人信息。通过优先级的调整，可以改变任务的调度顺序。"

### 3.5 用户与权限 `/users`

**演示步骤**：
1. 展示用户列表（5 个用户，含角色、部门）
2. 点击某个用户行，展开权限详情

## 5. 演示话术模板

### 5.1 开场白

> "欢迎各位领导和专家。今天的演示是一个前端 Demo，数据全部为 Mock，不涉及真实的后端服务。我们将展示统一光盘库管理平台的核心功能，包括站点管理、统一检索、任务管理、权限管理和审计日志。"

### 5.2 功能演示过渡

> "关于 XXX 功能，这里需要说明一下：目前 Demo 阶段使用的是 Mock 数据，展示了交互逻辑。后续接入真实后端后，将支持完整的数据持久化和实时同步。"

### 5.3 回应"这是真实功能吗？"类问题

> "这部分是前端交互的 Demo，数据是从 Mock 数据中读取的。点击按钮会有 Toast 提示，列表也会响应，但数据不会真正保存。这是为了验证交互流程是否流畅。真实的数据存储和后端调度将在下一阶段实现。"

### 5.4 回应"为什么没有登录页？"类问题

> "Demo 阶段所有页面都可以直接访问，方便演示。真实生产环境会集成 ADFS 单点登录，需要凭据才能访问系统。"

### 5.5 回应"数据为什么不更新？"类问题

> "目前使用的是本地 Mock 数据，页面刷新后会恢复初始状态。后续接入 WebSocket 后，站点状态、任务进度、告警信息都会实时更新。"

| 异常 | 应对 |
|------|------|
| 页面白屏 | 检查控制台报错，确认路由正确 |
| 数据不显示 | 确认 import 路径正确，Mock 数据导出正常 |
