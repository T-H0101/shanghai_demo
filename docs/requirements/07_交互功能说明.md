# P0 阶段交互功能说明

本文档记录 P0 阶段已实现的全部前端交互功能，基于 mock 数据 + React state + Toast 反馈实现。

---

## 一、全局交互

### 1.1 通知中心（Header 通知图标）

**实现位置**：`components/dashboard/header.tsx`

**交互行为**：
| 操作 | 行为 |
|------|------|
| 点击通知图标 | 展开 DropdownMenu 通知面板 |
| 点击"全部已读" | 所有通知标记为已读，红点消失，显示 Toast |
| 点击单条通知 | 该条标记为已读，显示对应类型 Toast（info/warning/success/destructive） |
| 无未读通知时 | 红点不显示 |

**状态管理**：模块级单例 `store/notification.ts`，页面刷新后恢复初始状态

**Toast 示例**：
- 通知点击后：`{ title: "站点离线告警", description: "成都研发基地核心交换机无响应..." }`

---

### 1.2 Toast 反馈机制

**实现位置**：`hooks/use-toast.ts` + `components/ui/toast.tsx` + `components/providers.tsx`

**使用方式**：
```typescript
import { toast } from "@/hooks/use-toast"

toast({ title: "操作成功", description: "任务已暂停" })
toast({ title: "导出成功", description: "已导出 8 条记录", variant: "destructive" })
```

**已接入 Toast 的操作**：
- 通知中心：单条点击、全部已读
- 站点管理：同步、SSO 跳转、禁用/启用、新建站点
- 统一检索：检索完成、导出、发起回迁
- 任务管理：暂停、恢复、重试、重置、优先执行、新建任务
- 审计日志：数字签名校验、导出 Excel/CSV/JSON
- 系统设置：保存、重置、导出配置、测试告警邮件

| 进入 SSO 控制台（详情） | Toast 提示跳转中 → Toast 跳转成功 |

## 七、Mock Enterprise Authentication Demo

### 7.1 Site-based Mock Authentication Flow

**实现位置**：`app/login/page.tsx` + `lib/mock/auth.ts` + `lib/auth/session.ts`

**登录流程（增强后）**：
1. 输入域账号后，自动显示该账号 `allowedSites` 对应的站点 Badge 列表
2. Site 下拉框仅显示该账号被授权的站点（disabled 直到输入账号）
3. 输入密码，选择目标站点
4. 点击登录后有 1.5s loading 状态，显示 "Connecting to enterprise federation service..."
5. `validateMockCredentials()` 校验账号密码
6. `validateSiteAccess()` 校验站点权限
7. 通过后 `saveMockSession()` 写入 localStorage，跳转首页

**UI 交互增强**：
- 输入账号后：显示 "Available Sites" Badge 列表
- Site Select：根据 `allowedSites` 动态过滤，未输入账号时 disabled
- 登录按钮：loading 时显示 "Connecting to enterprise federation service..."

**Site 权限模型**：
| 账号 | 角色 | 可访问站点 |
|------|------|-----------|
| admin | 集团超级管理员 | 全部 6 个站点 |
| ops | 运维管理员 | 北京、广州 |
| audit | 审计管理员 | 南京、武汉 |
| operator | 站点操作员 | 成都、上海 |

**权限校验失败**：`No access permission for selected site`

### 7.2 路由守卫

**实现位置**：`components/auth/route-guard.tsx`（`AppShell` 间接引用）

- 未登录访问业务页 → 自动跳转 `/login`
- 已登录访问 `/login` → 自动跳转首页

### 7.3 会话存储

**localStorage 字段**：
| Key | 内容 |
|-----|------|
| `mock_token` | 模拟 token（格式：`mock_demo_{username}_{timestamp}`） |
| `mock_user` | 用户名 |

| `mock_role_level` | 角色级别（group_admin / site_admin / auditor / operator） |
| `mock_site` | 当前登录站点 |
| `mock_department` | 部门 |
| `mock_login_time` | 登录时间 |

### 7.4 Topbar 用户信息

**显示内容**：`{站点} · {角色} · {部门}`（企业控制台风格）

**示例**：`上海数据中心 · 集团超级管理员 · 信息技术部`

**Logout 按钮**：清除 localStorage，跳转 `/login`

### 7.5 声明

当前系统为 Mock Enterprise Authentication Demo，根据 `mockUsers.allowedSites` 模拟企业 Site 权限隔离逻辑，不包含真实 ADFS / LDAP / OAuth / JWT / SSO Federation 服务。

### 8.3 状态管理方案

- **页面级状态**：各页面使用 `useState` 管理本地数据（如站点列表、任务列表）
- **跨组件状态**：`store/notification.ts` 使用模块级单例 + `useEffect` 订阅模式，实现通知状态在 Header 中的共享
- **数据持久化**：当前均为内存 state，页面刷新后恢复 mock 初始值

### 8.4 Toast 使用规范

使用 `hooks/use-toast.ts` 提供的 `toast()` 函数：
- `toast({ title, description })` — 默认样式
- `toast({ title, description, variant: "destructive" })` — 错误/警告样式
