## 3. 页面职责说明

### 3.1 首页仪表盘（`/`）

**职责**：展示平台全局状态概览

**包含组件**：
- `StatsCards`：4 个统计卡片（站点总数/容量/活跃任务/告警）
- `SyncTrendChart`：同步趋势图表（Recharts）
- `SiteHealthHeatmap`：站点健康热力图
- `TaskTable`：实时任务流水线（3 条任务）
- `AlertCenter`：告警中心（未处理告警列表）

**数据来源**：静态 mock 数据（`components/dashboard/` 内嵌）

**设计依据**：企业级后台首页标准布局，统计卡片 + 图表 + 告警，符合运维控制台风格。

---

### 3.2 站点管理（`/sites`）

**职责**：统一管理各站点基础信息、状态、存储容量

**布局**：双栏布局（列表 + 详情侧栏）

**包含组件**：
- `StatCards`：4 个统计卡片（站点总数/同步中/平均存储/异常站点）
- `PageHeader`：标题 + 操作按钮
- `Table`：站点列表（名称/状态/IP/数据中心/联系人/设备/同步/存储/操作）
- `DetailPanel`：站点详情侧栏

**核心交互**：
- 站点搜索（关键词过滤）
- 状态下拉筛选
- 新建站点 Dialog
- 同步按钮（Toast + 更新同步时间）
- 禁用/启用切换
- SSO 跳转 Toast

**设计依据**：标准管理系统列表页 + 详情侧栏，兼容站点规模扩展。

---

### 3.3 盘架管理（`/racks`）

**职责**：展示盘架列表与槽位使用情况

**布局**：双栏布局（列表 + 详情侧栏）

**包含组件**：
- `StatCards`：统计卡片（盘架总数/正常/预警+故障/平均使用率）
- `Table`：盘架列表（盘架ID/站点/数据中心/盘笼/槽位/使用率/状态/同步时间）
- `DetailPanel`：盘架详情 + `SlotGrid` 槽位可视化
- `SlotGrid`：槽位网格（已占用/空闲状态展示）

**核心交互**：
- 盘架列表点击选中 → 详情侧栏
- 槽位可视化（蓝色已占用/灰色空闲）

**设计依据**：盘架是物理设备，槽位可视化有助于理解光盘库使用率。

---

### 3.4 任务管理（`/tasks`）

**职责**：统一管理备份/恢复/巡检/刻录任务

**布局**：双栏布局（任务列表 + 日志面板）

**包含组件**：
- `StatCards`：统计卡片（任务总数/运行中/已暂停/今日完成）
- `AlertCard`：异常任务告警卡片
- `Tabs`：任务类型切换（全部/备份/恢复/巡检/刻录）
- `Table`：任务列表（任务/类型/站点/优先级/进度/状态/操作）
- `DetailPanel`：实时任务日志

**核心交互**：
- 类型 Tab 切换
- 状态 + 关键词筛选
- 操作按钮（暂停/恢复/重试/重置）
- 新建任务 Dialog
- Toast 反馈

**设计依据**：任务管理是企业级系统的核心，实时日志面板有助于监控任务执行状态。

---

### 3.5 用户与权限（`/users`）

**职责**：管理用户账号与 RBAC 权限

**布局**：双栏布局（用户列表 + 权限详情）

**包含组件**：
- `StatCards`：统计卡片（用户总数/活跃/已锁定/待同步）
- `Table`：用户列表（用户/部门/角色/可访问站点/状态/最后登录/权限同步）
- `DetailPanel`：权限详情 Tabs（站点/设备/存储卷/任务/日志）

**核心交互**：
- 用户列表点击选中 → 权限详情
- 权限树 Switch（仅 UI，无保存逻辑）
- 全站点提醒 Switch

**设计依据**：RBAC 权限管理需要清晰的层级展示，Tab 分组有助于理解权限结构。

---

### 3.6 审计日志（`/logs`）

**职责**：日志查询、筛选、详情查看、导出

**布局**：单栏布局（日志表格 + 详情面板）

**包含组件**：
- `StatCards`：统计卡片（24h 日志量/成功率/安全事件/失败操作）
- `Tabs`：日志类型切换（操作流水/安全日志/系统日志/任务日志/合规报表/告警策略）
- `FilterRow`：筛选条件（类型/站点/操作人/结果/日期）
- `Table`：日志列表（8 列）
- `DetailPanel`：JSON 详情 + 签名状态 + API Trace Chain

**核心交互**：
- Tab 切换过滤
- 站点/操作人/结果筛选联动
- 日志行点击 → 详情面板
- 数字签名校验按钮
- 导出 Excel/CSV/JSON Toast

**设计依据**：审计日志是企业合规要求，JSON 详情和 API Trace 有助于问题排查。

---

### 3.7 统一检索（`/search`）

**职责**：跨站点全局文件检索

**布局**：单栏布局（筛选 + 结果表格）

**包含组件**：
- `NoticeCard`：权限可见范围提示
- `FilterCard`：高级筛选（站点/部门/文件类型/光盘编号/存储卷）
- `Table`：检索结果（文件名/路径/大小/站点/光盘编号/盘架/部门/创建时间）
- `Pagination`：分页控件

**核心交互**：
- 高级筛选 Select 联动过滤
- 关键词搜索
- 检索/重置按钮
- 导出 Toast
- 发起回迁 Toast

**设计依据**：检索是核心业务功能，需要清晰的筛选条件和结果展示。

---

### 3.8 系统设置（`/settings`）

**职责**：系统配置与服务监控

**布局**：双栏布局（设置卡片 + 服务监控）

**包含组件**：
- `PageHeader`：标题 + 操作按钮
- `SettingsCard` × 4：同步设置/告警设置/安全设置/任务设置
- `ServiceCard` × 6：服务监控卡片（API Gateway/PostgreSQL/Sync Service/等）

**核心交互**：
- Switch/Input/Select 实时修改
- 保存/重置/导出/测试告警按钮
- Toast 反馈

**设计依据**：配置页面需要分区展示，服务监控卡片展示基础设施健康状态。

## 5. 设计依据

### 5.1 企业级后台风格

- **配色**：以 Slate 灰蓝为主色调，蓝色为强调色，符合企业软件低调专业风格
- **布局**：侧边栏 + 顶栏 + 内容区，符合 SaaS 控制台标准布局
- **组件**：使用 Radix UI 基础组件，交互一致，行为可预测
- **表格**：紧凑行列设计，支持 hover 和选中状态，适合数据密集型展示

### 5.2 信息密度

- **统计卡片**：每个页面顶部 4 个统计卡片，快速了解关键指标
- **双栏布局**：列表页采用左右分栏，左侧列表 + 右侧详情，节省空间同时保持信息完整
- **Tab 分组**：多类型数据（任务类型、日志类型、权限分组）使用 Tab 切换，避免页面臃肿

### 5.3 交互设计

- **即时反馈**：所有操作按钮有 Toast 反馈，让用户知道操作结果
- **状态可视化**：进度条、状态 Badge、颜色编码，让状态一目了然
- **筛选联动**：筛选条件实时联动结果，减少用户操作步骤
