# 测试指南

> 本文档说明如何使用当前 Demo 进行功能测试，记录测试步骤和预期结果。
> 测试范围：系统定位 + 核心设计原则相关的核心功能点。

---

## 一、测试准备

### 1.1 环境要求

| 环境 | 要求 |
|------|------|
| 浏览器 | Chrome/Firefox/Edge 最新版 |
| 分辨率 | ≥1920×1080 |
| Node.js | ≥18.17.0 |
| 运行命令 | `npm run dev` |

### 1.2 测试账号

| 账号 | 密码 | 角色 | 可访问站点 |
|------|------|------|-----------|
| admin | admin | 集团超级管理员 | 全部 6 个站点 |
| ops | ops | 运维管理员 | 北京、广州 |
| audit | audit | 审计管理员 | 南京、武汉 |
| operator | operator | 站点操作员 | 成都、上海 |

### 1.3 测试入口

1. 访问 `http://localhost:3000/login`
2. 使用上述账号密码登录
3. 登录成功后进入控制台

### 3.1 Mock 实现完成的功能点

以下功能已通过 Mock 数据和前端逻辑实现，测试可直接验证：

| 功能点 | 实现说明 | 测试验证方式 |
|--------|---------|-------------|
| 任务下发机制 | 新建任务时状态机：`pending_dispatch → dispatched → running` | 新建任务后观察状态变化和控制台输出 |
| 集团账号权限限制 | admin 账号不能发起刻录/回迁任务 | 使用 admin 登录后尝试创建刻录/回迁任务 |
| 站点数据独立性 | 各站点数据通过 siteId 关联，无直接耦合 | 查看不同站点数据，相互独立 |
