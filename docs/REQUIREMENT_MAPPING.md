# 需求覆盖分析报告

## 文档信息

- 源文件：`docs/source/requirements.md`
- 分析目标：建立「需求文档 ↔ 当前 Demo 实现」的映射关系
- 分析方式：静态代码扫描 + 需求文档比对

## 一、需求模块总览

需求规格说明书包含以下 6 大模块：

| 模块编号 | 模块名称 | 包含功能 |
|----------|----------|----------|
| 一 | 整体架构与核心原则 | 系统定位、核心设计原则 |
| 二 | 基础支撑模块 | 站点管理、统一身份认证、数据同步 |
| 三 | 核心管控模块 | 账号管理、账号权限分配、部门管理 |
| 四 | 业务操作模块 | 统一检索、统一任务管理、盘笼统一管理 |
| 五 | 辅助保障模块 | 日志管理、光盘索引导出 |
| 六 | 非功能需求 | 性能、安全、兼容性、可维护性 |

## 六、Demo 阶段建议实现范围

### 6.1 优先级 P0（核心演示必需）

> 不影响 mock 数据和 UI 风格，重点补全交互闭环

1. **站点管理**：站点搜索过滤联动、新增站点 Dialog 表单（名称/编码/数据中心/IP/端口/联系人）
2. **任务管理**：新建任务 Dialog 表单（任务类型/站点/优先级/光盘编号）；任务操作按钮（暂停/恢复）增加 Toast 反馈
3. **统一检索**：高级筛选 Select 与 mock 数据真实联动；分页逻辑修复
4. **审计日志**：筛选器（站点/操作人/结果）与 mock 数据真实联动；日志详情 JSON 复制功能
5. **用户与权限**：权限树 Switch 修改后增加 Toast 反馈（权限修改需同步提示）

### 6.2 优先级 P1（提升演示效果）

> 增加真实感，优化数据展示

1. **系统设置**：「保存」按钮增加成功提示；「重置」恢复默认值
2. **日志导出**：模拟导出（生成带随机数据的文本文件下载）
3. **盘架管理**：搜索过滤联动；移位 Dialog 表单
4. **站点切换**：SSO 按钮点击模拟跳转（新窗口打开目标 URL + Toast 提示已跳转）

### 6.3 优先级 P2（增强功能，可选）

> Demo 演示可选增强项

1. **登录页**：简单登录表单（用户名/密码），登录成功后显示所有页面（无真实验证）
2. **首页仪表盘**：统计卡片数值模拟定时小幅波动（useEffect + setInterval）
3. **告警中心**：告警卡片点击跳转到对应详情页
4. **任务日志导出**：模拟导出任务日志

### 6.4 不建议在 Demo 阶段实现

以下需求涉及后端架构，Demo 阶段不建议引入复杂度：

- Mock API 层（async/await 模拟）
- 状态管理库（Zustand/Jotai）
- 登录认证（ADFS 集成）
- 实时数据轮询（WebSocket / SSE）
- 后台任务（导出异步执行）
- 邮件/通知推送
