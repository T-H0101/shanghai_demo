## 一、项目概述

### 1.1 核心功能

系统名称为「档案数据长久保存 异构异地备份与灾难恢复系统」，是一套面向档案馆/企事业单位的档案数据全生命周期管理平台。核心功能包括：

- **数据接收**：支持硬盘数据接收、光盘数据接收、在线数据接收（CIFS/NFS 网络挂载）、离线接收等多种数据入库方式
- **数据管理**：对已入库档案数据包进行统一管理与检索
- **利用查询**：提供档案数据的检索、浏览、导出等利用服务
- **备份与恢复**：支持异构异地备份与灾难恢复
- **数据同步存档**：跨节点/跨介质的数据同步与长期保存
- **智能分析**：对档案数据进行智能分析处理

### 1.2 平台要求

- **运行环境**：Linux 服务器（演示环境基于 Debian/Ubuntu 系，可见 `/run`、`/dev/shm`、`/media/dsk1`、`/boot`、`/data` 等标准 Linux 目录结构）
- **客户端**：Web 浏览器访问（B/S 架构）
- **存储介质**：支持本地磁盘、移动硬盘、网络存储（CIFS/NFS 协议挂载）、蓝光光盘等

### 1.3 系统定位

本系统定位为档案行业专用的数据长期保存与灾备平台，解决档案馆在数字化过程中面临的数据接收、多介质备份、异地灾备、数据包管理等核心痛点。

## 三、数据接收模块

> 本次演示聚焦于「在线数据接收」和「离线接收任务」两个子模块。

### 3.1 在线数据接收

#### 3.1.1 页面布局

页面采用三栏 + 底部布局：

- **左侧面板** — 数据源区（树形目录选择器）
- **中间面板** — 文件列表（名称 / 类型 / 大小 / 日期，支持分页）
- **右侧面板** — 数据接收配置表单
- **底部面板** — 待接收数据列表

#### 3.1.2 数据源区

以树形结构展示系统可访问的所有存储路径，包括：

- 本地分区：`/run`、`/dev/shm`、`/run/lock`、`/boot`、`/boot/efi`、`/data`、`/run/user/1000`
- 本地文件夹：`/media/dsk1`
- 网络挂载目录：`/netshare/229fengbao`、`/netshare/10.188.1.187`
- **支持动态挂载新数据源**（参见「设备管理」章节的挂载异义盘功能）

选中数据源节点后，中间面板展示该路径下的文件列表。

#### 3.1.3 文件列表面板

| 字段 | 说明 |
|------|------|
| 名称 | 文件/文件夹名称 |
| 类型 | 分区 / 文件夹 / 文件 / ISO 镜像 |
| 大小 | 文件大小（文件夹显示 "-"，分区显示 "-"） |
| 日期 | 修改日期时间，格式 `YYYY-MM-DD HH:MM:SS` |

支持分页，默认每页 N 条，底部显示 "共 X 条 [页码]"。

#### 3.1.4 数据接收配置表单

用于填写数据包的元数据信息（在正式入库前配置）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 打包 csv 文件 | 输入框 + 选择按钮 | 是 | 点击弹出「选择 csv 文件」弹窗，从文件系统中选择数据包描述 csv 文件 |
| 备份单位 | 输入框（自动填充） | - | 从 csv 中自动解析 |
| 备份名称 | 输入框（自动填充） | - | 从 csv 中自动解析 |
| 备份编号 | 输入框（自动填充） | 是 | 从 csv 中自动解析 |
| 备份时间 | 输入框（自动填充） | - | 从 csv 中自动解析 |
| 备份说明 | 输入框（自动填充） | - | 从 csv 中自动解析（通常为原始数据路径） |
| 数据分类 | 下拉选择 | - | 可选值如：「借阅」等 |
| 包数量 | 输入框（自动填充） | - | 从 csv 中自动解析 |
| 存储路径 | 输入框 + 选择按钮 | 是 | 点击弹出「选择存储路径」弹窗，选择目标存储卷/路径 |

**自动填充机制**：选择 csv 文件后，系统自动解析 csv 内容并填充上述字段（备份单位、备份名称、备份编号、备份时间、备份说明、包数量），减少手动录入。

#### 3.1.5 csv 文件格式推断

从截图可见 csv 文件名示例：`2025年6月宁波市档案馆移交数据.csv`（824B）。csv 内容至少包含：备份单位、备份名称、备份编号、备份时间、备份说明、包数量。

#### 3.1.6 待接收数据面板

| 字段 | 说明 |
|------|------|
| 名称 | 待接收文件/文件夹名称 |
| 类型 | 文件/文件夹 |
| 来源 | 如「服务器端存储」 |
| 路径 | 完整存储路径 |

系统识别到待接收数据后自动添加到该列表，初始状态显示「暂无数据」。

## 四、设备管理模块（挂载异义盘）

> "异义盘"即外部存储设备/网络存储的挂载管理功能。

### 4.1 功能入口

在线数据接收页面的数据源区中触发（支持将移动盘/网络盘自动挂载到系统）。

### 4.2 挂载异义盘弹窗

#### 4.2.1 表单字段

| 字段 | 类型 | 必填 | 说明/示例值 |
|------|------|------|-------------|
| 设备名称 | 输入框 | 是 | 如 `172.168.6.15`（IP 地址） |
| 设备组 | 输入框 | 是 | 如「网盘」 |
| 协议类型 | 输入框 | 是 | 如 `CIFS` |
| 编码类型 | 按钮选择 | - | 如 `UTF-8` |
| 挂载目录 | 输入框 | 是 | 如 `/netshare/172.168.6.15/` |
| 管理目录 | 输入框 | 是 | 如 `/netshare/172.168.6.15/` |
| 获取 | 按钮 | - | 点击获取远程共享列表 |
| 数据源 | 输入框 | 是 | 如 `\\172.168.6.15\public`（Windows 共享路径格式） |
| 用户 | 输入框 | - | CIFS 认证用户名 |
| 密码 | 输入框 | - | CIFS 认证密码 |
| 权限 | 单选 + 按钮 | - | 单选「只读」；按钮「权限测试」 |
| 限速 | 输入框 | - | 单位：MB/s |
| 附加参数 | 输入框 | - | 如 `vers=2.0`（CIFS 协议版本参数） |

#### 4.2.2 交互流程

1. 填写设备名称、设备组、协议类型、挂载目录等必要信息
2. 点击「获取」按钮，系统自动发现远程共享（数据源字段自动填充）
3. 填写认证信息（用户/密码）
4. 点击「权限测试」验证读写权限
5. 提交挂载，日志区域实时显示挂载进度（如 `172.168.6.15 正在初始化`）
6. 挂载成功后，数据源区树形目录自动新增对应节点

#### 4.2.3 支持的协议

- **CIFS/SMB**：用于挂载 Windows 共享目录或 NAS 设备（示例中使用 `vers=2.0`）
- 理论上可扩展支持 NFS 等协议

### 4.3 存储路径选择弹窗

在数据接收配置中点击存储路径选择按钮触发。

- 以树形视图展示可用存储卷（如「蓝盘卷242」）
- 子节点按编号命名（`/001-iso` ~ `/018-iso` 等），推测对应蓝光盘槽位
- 支持分页（共 N 条）
- 选中后点击确定，存储路径自动填入表单

## 五、关键交互流程

### 5.1 在线接收数据包完整流程

```
挂载异义盘（如需要）
    → 在数据源区选择远程挂载目录
    → 中间面板浏览文件列表
    → 点击「打包 csv 文件」选择按钮
    → 弹出文件选择弹窗，选中对应的 .csv 文件
    → 系统自动解析 csv 并填充表单字段
    → 选择「数据分类」下拉项
    → 点击「存储路径」选择按钮
    → 弹出存储路径弹窗，选择目标存储卷
    → 确认提交，开始数据接收
    → 可在「离线接收任务」中查看任务进度
```

### 5.2 离线接收任务管理

- 在离线接收任务页面可查看所有接收任务的执行状态
- 支持按时间范围和备份编号筛选
- 任务状态包括「加载中...」（执行中）和「已完成」
- 已完成任务展示完成时间、文件数量等统计信息
