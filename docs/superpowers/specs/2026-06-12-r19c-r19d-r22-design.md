## 3. R.19C 数据流 (package push, 60s)

### 3.1 Agent 端 (Sync Pusher)

```
T+0s:    Pusher 触发 (60s 周期)
T+0.1s:  PG reader 增量查:
           - tbl_task WHERE update_dt > last_watermark
           - tbl_disc WHERE update_dt > last_watermark
           - tbl_logical_volume WHERE update_dt > last_watermark
           - tbl_magzines WHERE update_dt > last_watermark
           - tbl_hd_info WHERE update_dt > last_watermark
           - tbl_disc_lib WHERE update_dt > last_watermark
T+0.5s:  生成 package JSON:
           {
             siteCode, batchId, sentAt,
             tables: [
               {name: "tbl_task", rows: [...], watermark: "..."},
               {name: "tbl_disc", rows: [...]}
             ]
           }
T+0.6s:  POST /api/site-agent/packages + HMAC-SHA256 头
         (复用 R.19A hmac.ts)
T+1s:    总控返 200 {batchId, accepted: N, rejected: M, errors: [...]}
T+1.1s:  Agent 更新 SQLite watermark
T+1.2s:  失败行入重试队列 (下个 60s 周期重试, 3 次后 dead-letter)
```

### 3.2 总控端 (POST /api/site-agent/packages)

- 验签 (HMAC, R.19A 已有)
- 入站 `sync_package_log` (R.2H.3 已有, 复用)
- inlineUpsert (R.2H.2 已有, 复用) 各小表 → `unified_*`
- 写 `audit_log` (R.4.8 已有)
- 返 `{batchId, accepted, rejected, errors}`

### 3.3 R.19C 新增文件

- `app/api/site-agent/packages/route.ts` (总控端, 新)
- `scripts/agent/lib/sync-pusher.ts` (Agent 端, 新)
- `scripts/agent/lib/pg-reader.ts` (Agent 端, 新, 增量读 6 小表)

### 3.4 R.19C 复用

- 中心 `sync_package_log` 表
- 中心 `inlineUpsert` (Sprint 2H.2)
- Agent `hmac.ts` (R.19A)

## 4. R.19D 数据流 (control poll, 5s)

### 4.1 Agent 端 (Ctrl Poller)

```
T+0s:    Poller 触发 (5s 周期)
T+0.1s:  GET /api/control/commands/pending?siteCode=SH01
T+0.3s:  总控返 [{id, commandType, targetId, targetType, payload}, ...]
         (只返 status='pending' 且 source_site_id 匹配)
T+0.4s:  Agent 检查 SQLite: commandId 是否已 ack (dedupe)
T+0.5s:  新命令 → POST /api/control/commands/[id]/ack (status='pulled')
T+0.6s:  命令入本地执行队列 (按 commandType 分发)
T+0.7s:  PG adapter 执行 (R.22, 见 §5)
T+1s:    Agent POST /api/control/commands/[id]/result {
           status: "success" | "unsupported" | "failed",
           result: {affectedRows, before, after, blocker?, reason?},
           errorMessage: null
         }
T+1.1s:  总控写 control_command.status + result + audit_log
```

## 5. R.22 PG Action Adapter

### 5.1 reset 路径 (R.18 capability matrix 决议)

| task_type | 含义 (docx) | reset 路径 | 依据 |
|---|---|---|---|
| 0 (备份任务) | 备份/刻录 | ✅ UPDATE status=1, burn_status=0 | R.4 历史, docx 推断 status=1=数据准备中 |
| 1 (恢复任务) | 数据回迁 | ❌ unsupported, reason="回迁类无 reset 语义, status=1 官方=开始回迁" | docx L135 |
| 2 (刻录并封盘) | 刻录封盘 | ✅ UPDATE status=1, burn_status=0 | 同上 |
| 3 (EPSON 接口) | EPSON 出版 | ✅ UPDATE status=1, burn_status=0 | 同上 |
| 4 (扫描任务) | 扫描入库 | ✅ UPDATE status=1, burn_status=0 | 同上 |
| 其他/未知 | — | ❌ unsupported, reason="task_type 未知" | 安全降级 |

**注**: 备份/刻录/扫描类 (type ∈ {0,2,3,4}) 的 reset 写 `status=1, burn_status=0` 是 **R.4 历史推断**, **非官方 reset 协议**. toast 明标"R.4 推断, 官方未定义 reset, 待站点提供正式协议后调整".

## 6. 错误处理

| 场景 | 处理 |
|---|---|
| Agent 推 package 失败 (HTTP 5xx, 网络断) | SQLite 重试队列, 下个 60s 周期重试, 3 次失败入 dead-letter |
| Agent 拉 control 失败 | 同上, 下个 5s 周期重试 |
| Agent 推 package 含 stale 行 (unified_* 已有, 源端 update_dt 落后) | inlineUpsert ON CONFLICT UPDATE, 幂等 |
| Agent 同命令重复 ack | commandId 唯一索引, dedupe |
| Agent 推的 rows 包含 schema 不匹配字段 | 总控单行拒绝, 返回 rejected list, Agent 记录 |
| Agent PG adapter 失败 (连接断, 表不存在) | command 返 failed + errorMessage, 总控 UI 显示 "agent 失败" |
| Agent 进程崩溃 (heartbeat 过期) | 总控 /sync 显示 expired, R.19A 已有 |
| Agent reset type=1 | 返 unsupported + reason, 不写 status=1 |
